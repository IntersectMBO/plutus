   39 {- Note [CSE]
   40 
   41 -------------------------------------------------------------------------------
   42 1. Simplifications
   43 -------------------------------------------------------------------------------
   44 
   45 This is a simplified (i.e., not fully optimal) implementation of CSE. The one simplification
   46 we made is:
   47 
   48 - No alpha equivalence check, i.e., `\x -> x` and `\y -> y` are considered different expressions.
   49   Moreover, subterms that have lambda abstractions inside are never considered equal, because
   50   the renamer runs at the start of the pass, giving each lambda a unique name.
   51 
   52 -------------------------------------------------------------------------------
   53 2. How does it work?
   54 -------------------------------------------------------------------------------
   55 
   56 We use the following example to explain how the implementation works:
   57 
   58 \x y -> (1+(2+x))
   59         +
   60         (case y [ (1+(2+x)) + (3+x)
   61                 , (2+x) + (3+x)
   62                 , 4+x
   63                 ]
   64         )
   65 
   66 The implementation makes several passes on the given term. The first pass collects builtin
   67 arity information as described above.
   68 
   69 In the second pass, we assign a unique ID to each `LamAbs`, `Delay`, and each `Case` branch.
   70 Then, we annotate each subterm with a path, consisting of IDs encountered from the root
   71 to that subterm (not including itself). The reason to do this is because `LamAbs`, `Delay`,
   72 and `Case` branches represent places where computation stops, i.e., subexpressions are not
   73 immediately evaluated, and may not be evaluated at all.
   74 
   75 In the above example, the ID of `\x` is 0, the ID of `\y` is 1, and the IDs of the
   76 three case branches are 2, 3, 4 (the actual numbers don't matter, as long as they are unique).
   77 The path for the first `1+(2+x)` and the first `2+x` is "0.1"; the path for the second
   78 `1+(2+x)` and the second `2+x` is "0.1.2"; the path for `4+x` is "0.1.4".
   79 
   80 In the third pass, we calculate a count for each (term, path). Depending on the `cseWhichSubterms`
   81 option, `term` can be any subterm or a subterm that is not work-free, while `path` is the
   82 path of that term. If the same term has two paths, and one is an ancestor (i.e., prefix)
   83 of the other, we increment the count for the ancestor path in both instances.
   84 
   85 In the above example, there are three occurrences of `2+x`, whose paths are "0.1", "0.1.2"
   86 and "0.1.3", respectively. The first path is an ancestor of the latter two. Therefore,
   87 the count for `(2+x, "0.1")` is 3, while the count for `(2+x, "0.1.2")` and `(2+x, "0.1.3")`
   88 is 0. The following all have a count of 1: `(3+x, "0.1.2")`, `(3+x, "0.1.3")` and
   89 `(4+x, "0.1.4")`.
   90 
   91 Now, each `(term, path)` pair whose count is greater than 1 is a CSE candidate.
   92 In the above example, the CSE candidates are `(2+x, "0.1")` and `(1+(2+x), "0.1")`, assuming
   93 we are excluding work-free subterms during this pass. Note that `3+x` is not a CSE candidate,
   94 because it has two paths, and neither has a count greater than 1. `2+` is also not a CSE candidate,
   95 because it is workfree.
   96 
   97 The CSE candidates are then processed in descending order of their `termAstSize`s. For each CSE
   98 candidate, we generate a fresh variable, create a LamAbs for it under its path, and substitute
   99 it for all occurrences in the original term whose paths are descendents (or self) of
  100 the candidate's path. The order is because a bigger expression may contain a small subexpression.
  101 
  102 In the above example, we first process CSE candidate `(1+(2+x), "0.1")`. We create a fresh
  103 variable `cse1` for it, perform substitution, and create a `LamAbs` under path "0.1" (i.e., around
  104 the body of `y`). After processing this CSE candidate, the original term becomes
  105 
  106 \x y -> (\cse1 -> cse1
  107                   +
  108                   (case y [ cse1 + (3+x)
  109                           , (2+x) + (3+x)
  110                           , 4+x
  111                           ]
  112         ) (1+(2+x))
  113 
  114 The second CSE candidate is processed similarly, and the final result is
  115 
  116 \x y -> (\cse2 -> (\cse1 -> cse1
  117                             +
  118                             (case y [ cse1 + (3+x)
  119                                     , cse2 + (3+x)
  120                                     , 4+x
  121                                     ]
  122                   ) (1+cse2)
  123         ) (2+x)
  124 
  125 Here's another example:
  126 
  127 force (force ifThenElse
  128          (lessThanEqualsInteger 0 0)
  129          (delay ((1+2) + (1+2)))
  130          (delay (1+2))
  131       )
  132 
  133 In this case, the first two occurrences of `1+2` can be CSE'd, but the third occurrence
  134 can not. This is ensured by checking the path when substituting `cse1` for `1+2`. The result is
  135 
  136 force (force ifThenElse
  137          (lessThanEqualsInteger 0 0)
  138          (delay ((\cse1 -> cse1 + cse1) (1+2))
  139          (delay (1+2))
  140       )
  141 
  142 -------------------------------------------------------------------------------
  143 3. When should CSE run?
  144 -------------------------------------------------------------------------------
  145 
  146 CSE should run for multiple iterations, and should interleave with inlining. The following
  147 example illustrates why:
  148 
  149 \x ->
  150   f
  151     ((\y -> 1+(y+y)) (0+x))
  152     ((\z -> 2+(z+z)) (0+x))
  153 
  154 There is no inlining opportunity in this term. After the first iteration of CSE, where
  155 the common subepxression is `0+x`, we get:
  156 
  157 \x ->
  158   (\cse1 ->
  159     f
  160       ((\y -> 1+(y+y)) cse1)
  161       ((\z -> 2+(z+z)) cse1)
  162   ) (0+x)
  163 
  164 Now `y` and `z` can be inlined, after which we get
  165 
  166 \x ->
  167   (\cse1 ->
  168     f
  169       (1+(cse1+cse1))
  170       (2+(cse1+cse1))
  171   ) (0+x)
  172 
  173 Now there's a new common subexpression: `cse1+cse1`. So another iteration of CSE is
  174 needed, yielding:
  175 
  176 \x ->
  177   (\cse1 ->
  178     (\cse2 ->
  179       f
  180         (1+cse2)
  181         (2+cse2)
  182     ) (cse1+cse1)
  183   ) (0+x)
  184 
  185 With this example in mind, one may be tempted to make CSE part of the simplifier, and simply
  186 run it along with the rest of the simplifier. That is, however, a bad idea. CSE does the reverse
  187 of inlining; inlining tends to expose more optimization opportunities, and conversely, CSE
  188 tends to destroy optimization opportunities. Running CSE on a not-fully-optimized program
  189 may cause many optimization opportunities to be permanently lost. Give it a try if you want
  190 to see how bad it is!
  191 
  192 Therefore, this is what we do: first run the simplifier iterations. Then, run the CSE iterations,
  193 interleaving with the simplifier. For example, suppose max-simplifier-iterations-uplc=12, and
  194 max-cse-iterations=4. We first run 12 iterations of the simplifier, then run 4 iterations
  195 of CSE, with a simplifier pass after each iteration of CSE (i.e., the simplifier is run for a
  196 total of 16 times).
  197 
  198 Finally, since CSE can change the order or the number of occurrences of effects, it is only run
  199 when conservative optimization is off.
  200 -}

  215 data CseCandidate uni fun ann = CseCandidate
  216   { ccFreshName :: Name
  217   , ccTerm :: Term Name uni fun ()
  218   , ccAnnotatedTerm :: Term Name uni fun (Path, ann)
  219   {-^ `ccTerm` is needed for equality comparison, while `ccAnnotatedTerm` is needed
  220   for the actual substitution. They are always the same term barring the annotations. -}
  221   }
  222 
  223 cse
  224   :: ( MonadQuote m
  225      , Hashable (Term Name uni fun ())
  226      , Rename (Term Name uni fun ann)
  227      , ToBuiltinMeaning uni fun
  228      )
  229   => CseWhichSubterms
  230   -> BuiltinSemanticsVariant fun
  231   -> Term Name uni fun ann
  232   -> SimplifierT Name uni fun ann m (Term Name uni fun ann)
  233 cse whichSubterms builtinSemanticsVariant t0 = do
  234   t <- rename t0
  235   let annotated = annotate t
  236       commonSubexprs =
  237         -- Processed the common subexpressions in descending order of `termAstSize`.
  238         -- See Note [CSE].
  239         sortOn (Down . termAstSize)
  240           . fmap snd3
  241           -- A subexpression is common if the count is greater than 1.
  242           . filter ((> 1) . thd3)
  243           . join
  244           . Map.elems
  245           $ countOccs whichSubterms builtinSemanticsVariant annotated
  246   result <- mkCseTerm commonSubexprs annotated
  247   recordSimplification t0 CSE result
  248   return result
  249 
  250 -- | The second pass. See Note [CSE].
  251 annotate :: Term name uni fun ann -> Term name uni fun (Path, ann)
  252 annotate = flip evalState 0 . flip runReaderT [] . go
  253   where
  254     -- The integer state is the highest ID assigned so far.
  255     -- The reader context is the current path.
  256     go :: Term name uni fun ann -> ReaderT Path (State Int) (Term name uni fun (Path, ann))
  257     go t = do
  258       path <- ask
  259       case t of
  260         Apply ann fun arg -> Apply (path, ann) <$> go fun <*> go arg
  261         Force ann body -> Force (path, ann) <$> go body
  262         Constr ann i args -> Constr (path, ann) i <$> traverse go args
  263         Constant ann val -> pure $ Constant (path, ann) val
  264         Error ann -> pure $ Error (path, ann)
  265         Builtin ann fun -> pure $ Builtin (path, ann) fun
  266         Var ann name -> pure $ Var (path, ann) name
  267         LamAbs ann n body -> do
  268           freshId <- (+ 1) <$> lift get
  269           lift $ put freshId
  270           LamAbs (path, ann) n <$> local (freshId :) (go body)
  271         Delay ann body -> do
  272           freshId <- (+ 1) <$> lift get
  273           lift $ put freshId
  274           Delay (path, ann) <$> local (freshId :) (go body)
  275         Case ann scrut branches ->
  276           Case (path, ann)
  277             <$> go scrut
  278             <*> ( for branches $ \br -> do
  279                     freshId <- (+ 1) <$> lift get
  280                     lift $ put freshId
  281                     local (freshId :) (go br)
  282                 )

  284 {-| The notion of work-free expressions is extended to also include partially
  285 applied/forced built-ins. -}
  286 isWorkFree'
  287   :: forall name uni fun a
  288    . ToBuiltinMeaning uni fun
  289   => BuiltinSemanticsVariant fun
  290   -> Term name uni fun a
  291   -> Bool
  292 isWorkFree' builtinSemanticsVariant term =
  293   isWorkFree builtinSemanticsVariant term
  294     || not (isBuiltinSaturated term)
  295     || isForcingBuiltin term
  296   where
  297     isBuiltinSaturated =
  298       splitApplication >>> \case
  299         (Builtin _ fun, args) ->
  300           length args >= length (builtinArity (Proxy @uni) builtinSemanticsVariant fun)
  301         _term -> True
  302 
  303     isForcingBuiltin = \case
  304       Builtin {} -> True
  305       Force _ t -> isForcingBuiltin t
  306       _ -> False
  307 
  308 -- | The third pass. See Note [CSE].
  309 countOccs
  310   :: forall name uni fun ann
  311    . (Hashable (Term name uni fun ()), ToBuiltinMeaning uni fun)
  312   => CseWhichSubterms
  313   -> BuiltinSemanticsVariant fun
  314   -> Term name uni fun (Path, ann)
  315   -> HashMap (Term name uni fun ()) [(Path, Term name uni fun (Path, ann), Int)]
  316   {-^ Here, the value of the inner map not only contains the count, but also contains
  317   the annotated term, corresponding to the term that is the key of the outer map.
  318   The annotated terms need to be recorded since they will be used for substitution. -}
  319 countOccs whichSubterms builtinSemanticsVariant =
  320   foldrOf
  321     termSubtermsDeep
  322     (case whichSubterms of AllSubterms -> addToMap; ExcludeWorkFree -> addOrSkip)
  323     Map.empty
  324   where
  325     addOrSkip
  326       , addToMap
  327         :: Term name uni fun (Path, ann)
  328         -> HashMap (Term name uni fun ()) [(Path, Term name uni fun (Path, ann), Int)]
  329         -> HashMap (Term name uni fun ()) [(Path, Term name uni fun (Path, ann), Int)]
  330 
  331     addOrSkip t0
  332       | isWorkFree' builtinSemanticsVariant t0 = id
  333       | otherwise = addToMap t0
  334 
  335     addToMap t0 =
  336       Map.alter
  337         ( \case
  338             Nothing -> Just [(path, t0, 1)]
  339             Just paths -> Just $ combinePaths t0 path paths
  340         )
  341         t
  342       where
  343         t = void t0
  344         path = fst (termAnn t0)
  345 
  346 -- | Combine a new path with a number of existing (path, count) pairs.
  347 combinePaths
  348   :: forall name uni fun ann
  349    . Term name uni fun (Path, ann)
  350   -> Path
  351   -> [(Path, Term name uni fun (Path, ann), Int)]
  352   -> [(Path, Term name uni fun (Path, ann), Int)]
  353 combinePaths t path = go 1
  354   where
  355     go
  356       :: Int
  357       -> [(Path, Term name uni fun (Path, ann), Int)]
  358       -> [(Path, Term name uni fun (Path, ann), Int)]
  359     -- The new path is not a descendent-or-self of any existing path.
  360     go acc [] = [(path, t, acc)]
  361     go acc ((path', t', cnt) : paths)
  362       -- The new path is an ancestor-or-self of an existing path.
  363       -- Take over all counts of the existing path, remove the existing path,
  364       -- and continue.
  365       | path `isAncestorOrSelf` path' = go (acc + cnt) paths
  366       -- The new path is a descendent-or-self of an existing path.
  367       -- Increment the count for the existing path. There can only be one such
  368       -- existing path, so we don't need to recurse here.
  369       | path' `isAncestorOrSelf` path = (path', t', cnt + 1) : paths
  370       | otherwise = (path', t', cnt) : go acc paths

  372 mkCseTerm
  373   :: forall uni fun ann m
  374    . (MonadQuote m, Eq (Term Name uni fun ()))
  375   => [Term Name uni fun (Path, ann)]
  376   -> Term Name uni fun (Path, ann)
  377   -- ^ The original annotated term
  378   -> m (Term Name uni fun ann)
  379 mkCseTerm ts t = do
  380   cs <- traverse mkCseCandidate ts
  381   pure . fmap snd $ Foldable.foldl' (flip applyCse) t cs
  382 
  383 applyCse
  384   :: forall uni fun ann
  385    . Eq (Term Name uni fun ())
  386   => CseCandidate uni fun ann
  387   -> Term Name uni fun (Path, ann)
  388   -> Term Name uni fun (Path, ann)
  389 applyCse c = mkLamApp . transformOf termSubterms substCseVarForTerm
  390   where
  391     candidatePath = fst (termAnn (ccAnnotatedTerm c))
  392 
  393     substCseVarForTerm :: Term Name uni fun (Path, ann) -> Term Name uni fun (Path, ann)
  394     substCseVarForTerm t =
  395       if currTerm == ccTerm c && candidatePath `isAncestorOrSelf` currPath
  396         then Var (termAnn t) (ccFreshName c)
  397         else t
  398       where
  399         currTerm = void t
  400         currPath = fst (termAnn t)
  401 
  402     mkLamApp :: Term Name uni fun (Path, ann) -> Term Name uni fun (Path, ann)
  403     mkLamApp t
  404       | currPath == candidatePath =
  405           Apply
  406             (termAnn t)
  407             (LamAbs (termAnn t) (ccFreshName c) t)
  408             (ccAnnotatedTerm c)
  409       | currPath `isAncestorOrSelf` candidatePath = case t of
  410           Var ann name -> Var ann name
  411           LamAbs ann name body -> LamAbs ann name (mkLamApp body)
  412           Apply ann fun arg -> Apply ann (mkLamApp fun) (mkLamApp arg)
  413           Force ann body -> Force ann (mkLamApp body)
  414           Delay ann body -> Delay ann (mkLamApp body)
  415           Constant ann val -> Constant ann val
  416           Builtin ann fun -> Builtin ann fun
  417           Error ann -> Error ann
  418           Constr ann i ts -> Constr ann i (mkLamApp <$> ts)
  419           Case ann scrut branches -> Case ann (mkLamApp scrut) (mkLamApp <$> branches)
  420       | otherwise = t
  421       where
  422         currPath = fst (termAnn t)
  423 
  424 -- | Generate a fresh variable for the common subexpression.
  425 mkCseCandidate
  426   :: forall uni fun ann m
  427    . MonadQuote m
  428   => Term Name uni fun (Path, ann)
  429   -> m (CseCandidate uni fun ann)
  430 mkCseCandidate t = CseCandidate <$> freshName "cse" <*> pure (void t) <*> pure t
