   34 {- Note [CSE]
   35 
   36 -------------------------------------------------------------------------------
   37 1. Simplifications
   38 -------------------------------------------------------------------------------
   39 
   40 This is a simplified (i.e., not fully optimal) implementation of CSE. The one simplification
   41 we made is:
   42 
   43 - No alpha equivalence check, i.e., `\x -> x` and `\y -> y` are considered different expressions.
   44 
   45 -------------------------------------------------------------------------------
   46 2. How does it work?
   47 -------------------------------------------------------------------------------
   48 
   49 We use the following example to explain how the implementation works:
   50 
   51 \x y -> (1+(2+x))
   52         +
   53         (case y [ (1+(2+x)) + (3+x)
   54                 , (2+x) + (3+x)
   55                 , 4+x
   56                 ]
   57         )
   58 
   59 The implementation makes several passes on the given term. The first pass collects builtin
   60 arity information as described above.
   61 
   62 In the second pass, we assign a unique ID to each `LamAbs`, `Delay`, and each `Case` branch.
   63 Then, we annotate each subterm with a path, consisting of IDs encountered from the root
   64 to that subterm (not including itself). The reason to do this is because `LamAbs`, `Delay`,
   65 and `Case` branches represent places where computation stops, i.e., subexpressions are not
   66 immediately evaluated, and may not be evaluated at all.
   67 
   68 In the above example, the ID of `\x` is 0, the ID of `\y` is 1, and the IDs of the
   69 three case branches are 2, 3, 4 (the actual numbers don't matter, as long as they are unique).
   70 The path for the first `1+(2+x)` and the first `2+x` is "0.1"; the path for the second
   71 `1+(2+x)` and the second `2+x` is "0.1.2"; the path for `4+x` is "0.1.4".
   72 
   73 In the third pass, we calculate a count for each `(term, path)` pair, where `term` is a
   74 non-workfree term, and `path` is its path. If the same term has two paths, and one is an
   75 ancestor (i.e., prefix) of the other, we increment the count for the ancestor path in both
   76 instances.
   77 
   78 In the above example, there are three occurrences of `2+x`, whose paths are "0.1", "0.1.2"
   79 and "0.1.3", respectively. The first path is an ancestor of the latter two. Therefore,
   80 the count for `(2+x, "0.1")` is 3, while the count for `(2+x, "0.1.2")` and `(2+x, "0.1.3")`
   81 is 0. The following all have a count of 1: `(3+x, "0.1.2")`, `(3+x, "0.1.3")` and
   82 `(4+x, "0.1.4")`.
   83 
   84 Now, each `(term, path)` pair whose count is greater than 1 is a CSE candidate.
   85 In the above example, the CSE candidates are `(2+x, "0.1")` and `(1+(2+x), "0.1")`.
   86 Note that `3+x` is not a CSE candidate, because it has two paths, and neither has a count
   87 greater than 1. `2+` is also not a CSE candidate, because it is workfree.
   88 
   89 The CSE candidates are then processed in descending order of their `termAstSize`s. For each CSE
   90 candidate, we generate a fresh variable, create a LamAbs for it under its path, and substitute
   91 it for all occurrences in the original term whose paths are descendents (or self) of
   92 the candidate's path. The order is because a bigger expression may contain a small subexpression.
   93 
   94 In the above example, we first process CSE candidate `(1+(2+x), "0.1")`. We create a fresh
   95 variable `cse1` for it, perform substitution, and create a `LamAbs` under path "0.1" (i.e., around
   96 the body of `y`). After processing this CSE candidate, the original term becomes
   97 
   98 \x y -> (\cse1 -> cse1
   99                   +
  100                   (case y [ cse1 + (3+x)
  101                           , (2+x) + (3+x)
  102                           , 4+x
  103                           ]
  104         ) (1+(2+x))
  105 
  106 The second CSE candidate is processed similarly, and the final result is
  107 
  108 \x y -> (\cse2 -> (\cse1 -> cse1
  109                             +
  110                             (case y [ cse1 + (3+x)
  111                                     , cse2 + (3+x)
  112                                     , 4+x
  113                                     ]
  114                   ) (1+cse2)
  115         ) (2+x)
  116 
  117 Here's another example:
  118 
  119 force (force ifThenElse
  120          (lessThanEqualsInteger 0 0)
  121          (delay ((1+2) + (1+2)))
  122          (delay (1+2))
  123       )
  124 
  125 In this case, the first two occurrences of `1+2` can be CSE'd, but the third occurrence
  126 can not. This is ensured by checking the path when substituting `cse1` for `1+2`. The result is
  127 
  128 force (force ifThenElse
  129          (lessThanEqualsInteger 0 0)
  130          (delay ((\cse1 -> cse1 + cse1) (1+2))
  131          (delay (1+2))
  132       )
  133 
  134 -------------------------------------------------------------------------------
  135 3. When should CSE run?
  136 -------------------------------------------------------------------------------
  137 
  138 CSE should run for multiple iterations, and should interleave with inlining. The following
  139 example illustrates why:
  140 
  141 \x ->
  142   f
  143     ((\y -> 1+(y+y)) (0+x))
  144     ((\z -> 2+(z+z)) (0+x))
  145 
  146 There is no inlining opportunity in this term. After the first iteration of CSE, where
  147 the common subepxression is `0+x`, we get:
  148 
  149 \x ->
  150   (\cse1 ->
  151     f
  152       ((\y -> 1+(y+y)) cse1)
  153       ((\z -> 2+(z+z)) cse1)
  154   ) (0+x)
  155 
  156 Now `y` and `z` can be inlined, after which we get
  157 
  158 \x ->
  159   (\cse1 ->
  160     f
  161       (1+(cse1+cse1))
  162       (2+(cse1+cse1))
  163   ) (0+x)
  164 
  165 Now there's a new common subexpression: `cse1+cse1`. So another iteration of CSE is
  166 needed, yielding:
  167 
  168 \x ->
  169   (\cse1 ->
  170     (\cse2 ->
  171       f
  172         (1+cse2)
  173         (2+cse2)
  174     ) (cse1+cse1)
  175   ) (0+x)
  176 
  177 With this example in mind, one may be tempted to make CSE part of the simplifier, and simply
  178 run it along with the rest of the simplifier. That is, however, a bad idea. CSE does the reverse
  179 of inlining; inlining tends to expose more optimization opportunities, and conversely, CSE
  180 tends to destroy optimization opportunities. Running CSE on a not-fully-optimized program
  181 may cause many optimization opportunities to be permanently lost. Give it a try if you want
  182 to see how bad it is!
  183 
  184 Therefore, this is what we do: first run the simplifier iterations. Then, run the CSE iterations,
  185 interleaving with the simplifier. For example, suppose max-simplifier-iterations-uplc=12, and
  186 max-cse-iterations=4. We first run 12 iterations of the simplifier, then run 4 iterations
  187 of CSE, with a simplifier pass after each iteration of CSE (i.e., the simplifier is run for a
  188 total of 16 times).
  189 
  190 Finally, since CSE can change the order or the number of occurrences of effects, it is only run
  191 when conservative optimization is off.
  192 -}

  194 -- | In reverse order, e.g., "1.2.3" is `[3, 2, 1]`.
  195 type Path = [Int]
  196 
  197 isAncestorOrSelf :: Path -> Path -> Bool
  198 isAncestorOrSelf = isSuffixOf

  200 data CseCandidate uni fun ann = CseCandidate
  201   { ccFreshName     :: Name
  202   , ccTerm          :: Term Name uni fun ()
  203   , ccAnnotatedTerm :: Term Name uni fun (Path, ann)
  204   -- ^ `ccTerm` is needed for equality comparison, while `ccAnnotatedTerm` is needed
  205   -- for the actual substitution. They are always the same term barring the annotations.
  206   }
  207 
  208 cse ::
  209   ( MonadQuote m
  210   , Hashable (Term Name uni fun ())
  211   , Rename (Term Name uni fun ann)
  212   , ToBuiltinMeaning uni fun
  213   ) =>
  214   BuiltinSemanticsVariant fun ->
  215   Term Name uni fun ann ->
  216   SimplifierT Name uni fun ann m (Term Name uni fun ann)
  217 cse builtinSemanticsVariant t0 = do
  218   t <- rename t0
  219   let annotated = annotate t
  220       commonSubexprs =
  221         -- Processed the common subexpressions in descending order of `termAstSize`.
  222         -- See Note [CSE].
  223         sortOn (Down . termAstSize)
  224           . fmap snd3
  225           -- A subexpression is common if the count is greater than 1.
  226           . filter ((> 1) . thd3)
  227           . join
  228           . Map.elems
  229           $ countOccs builtinSemanticsVariant annotated
  230   result <- mkCseTerm commonSubexprs annotated
  231   recordSimplification t0 CSE result
  232   return result
  233 
  234 -- | The second pass. See Note [CSE].
  235 annotate :: Term name uni fun ann -> Term name uni fun (Path, ann)
  236 annotate = flip evalState 0 . flip runReaderT [] . go
  237   where
  238     -- The integer state is the highest ID assigned so far.
  239     -- The reader context is the current path.
  240     go :: Term name uni fun ann -> ReaderT Path (State Int) (Term name uni fun (Path, ann))
  241     go t = do
  242       path <- ask
  243       case t of
  244         Apply ann fun arg -> Apply (path, ann) <$> go fun <*> go arg
  245         Force ann body -> Force (path, ann) <$> go body
  246         Constr ann i args -> Constr (path, ann) i <$> traverse go args
  247         Constant ann val -> pure $ Constant (path, ann) val
  248         Error ann -> pure $ Error (path, ann)
  249         Builtin ann fun -> pure $ Builtin (path, ann) fun
  250         Var ann name -> pure $ Var (path, ann) name
  251         LamAbs ann n body -> do
  252           freshId <- (+ 1) <$> lift get
  253           lift $ put freshId
  254           LamAbs (path, ann) n <$> local (freshId :) (go body)
  255         Delay ann body -> do
  256           freshId <- (+ 1) <$> lift get
  257           lift $ put freshId
  258           Delay (path, ann) <$> local (freshId :) (go body)
  259         Case ann scrut branches ->
  260           Case (path, ann)
  261             <$> go scrut
  262             <*> ( for branches $ \br -> do
  263                     freshId <- (+ 1) <$> lift get
  264                     lift $ put freshId
  265                     local (freshId :) (go br)
  266                 )

  268 -- | The third pass. See Note [CSE].
  269 countOccs ::
  270   forall name uni fun ann.
  271   (Hashable (Term name uni fun ()), ToBuiltinMeaning uni fun) =>
  272   BuiltinSemanticsVariant fun ->
  273   Term name uni fun (Path, ann) ->
  274   -- | Here, the value of the inner map not only contains the count, but also contains
  275   -- the annotated term, corresponding to the term that is the key of the outer map.
  276   -- The annotated terms need to be recorded since they will be used for substitution.
  277   HashMap (Term name uni fun ()) [(Path, Term name uni fun (Path, ann), Int)]
  278 countOccs builtinSemanticsVariant = foldrOf termSubtermsDeep addToMap Map.empty
  279   where
  280     addToMap ::
  281       Term name uni fun (Path, ann) ->
  282       HashMap (Term name uni fun ()) [(Path, Term name uni fun (Path, ann), Int)] ->
  283       HashMap (Term name uni fun ()) [(Path, Term name uni fun (Path, ann), Int)]
  284     addToMap t0
  285       -- We don't consider work-free terms for CSE, because doing so may or may not
  286       -- have a size benefit, but certainly doesn't have any cost benefit (the cost
  287       -- will in fact be slightly higher due to the additional application).
  288       | isWorkFree builtinSemanticsVariant t0
  289         || not (isBuiltinSaturated t0)
  290         || isForcingBuiltin t0 =
  291           id
  292       | otherwise =
  293           Map.alter
  294             ( \case
  295                 Nothing -> Just [(path, t0, 1)]
  296                 Just paths -> Just $ combinePaths t0 path paths
  297             )
  298             t
  299       where
  300         t = void t0
  301         path = fst (termAnn t0)
  302 
  303     isBuiltinSaturated =
  304       splitApplication >>> \case
  305         (Builtin _ fun, args) ->
  306           length args >= length (builtinArity (Proxy @uni) builtinSemanticsVariant fun)
  307         _term -> True
  308 
  309     isForcingBuiltin = \case
  310       Builtin{} -> True
  311       Force _ t -> isForcingBuiltin t
  312       _ -> False

  314 -- | Combine a new path with a number of existing (path, count) pairs.
  315 combinePaths ::
  316   forall name uni fun ann.
  317   Term name uni fun (Path, ann) ->
  318   Path ->
  319   [(Path, Term name uni fun (Path, ann), Int)] ->
  320   [(Path, Term name uni fun (Path, ann), Int)]
  321 combinePaths t path = go 1
  322   where
  323     go ::
  324       Int ->
  325       [(Path, Term name uni fun (Path, ann), Int)] ->
  326       [(Path, Term name uni fun (Path, ann), Int)]
  327     -- The new path is not a descendent-or-self of any existing path.
  328     go acc [] = [(path, t, acc)]
  329     go acc ((path', t', cnt) : paths)
  330       -- The new path is an ancestor-or-self of an existing path.
  331       -- Take over all counts of the existing path, remove the existing path,
  332       -- and continue.
  333       | path `isAncestorOrSelf` path' = go (acc + cnt) paths
  334       -- The new path is a descendent-or-self of an existing path.
  335       -- Increment the count for the existing path. There can only be one such
  336       -- existing path, so we don't need to recurse here.
  337       | path' `isAncestorOrSelf` path = (path', t', cnt + 1) : paths
  338       | otherwise = (path', t', cnt) : go acc paths

  340 mkCseTerm ::
  341   forall uni fun ann m.
  342   (MonadQuote m, Eq (Term Name uni fun ())) =>
  343   [Term Name uni fun (Path, ann)] ->
  344   -- | The original annotated term
  345   Term Name uni fun (Path, ann) ->
  346   m (Term Name uni fun ann)
  347 mkCseTerm ts t = do
  348   cs <- traverse mkCseCandidate ts
  349   pure . fmap snd $ Foldable.foldl' (flip applyCse) t cs
  350 
  351 applyCse ::
  352   forall uni fun ann.
  353   (Eq (Term Name uni fun ())) =>
  354   CseCandidate uni fun ann ->
  355   Term Name uni fun (Path, ann) ->
  356   Term Name uni fun (Path, ann)
  357 applyCse c = mkLamApp . transformOf termSubterms substCseVarForTerm
  358   where
  359     candidatePath = fst (termAnn (ccAnnotatedTerm c))
  360 
  361     substCseVarForTerm :: Term Name uni fun (Path, ann) -> Term Name uni fun (Path, ann)
  362     substCseVarForTerm t =
  363       if currTerm == ccTerm c && candidatePath `isAncestorOrSelf` currPath
  364         then Var (termAnn t) (ccFreshName c)
  365         else t
  366       where
  367         currTerm = void t
  368         currPath = fst (termAnn t)
  369 
  370     mkLamApp :: Term Name uni fun (Path, ann) -> Term Name uni fun (Path, ann)
  371     mkLamApp t
  372       | currPath == candidatePath =
  373           Apply
  374             (termAnn t)
  375             (LamAbs (termAnn t) (ccFreshName c) t)
  376             (ccAnnotatedTerm c)
  377       | currPath `isAncestorOrSelf` candidatePath = case t of
  378           Var ann name            -> Var ann name
  379           LamAbs ann name body    -> LamAbs ann name (mkLamApp body)
  380           Apply ann fun arg       -> Apply ann (mkLamApp fun) (mkLamApp arg)
  381           Force ann body          -> Force ann (mkLamApp body)
  382           Delay ann body          -> Delay ann (mkLamApp body)
  383           Constant ann val        -> Constant ann val
  384           Builtin ann fun         -> Builtin ann fun
  385           Error ann               -> Error ann
  386           Constr ann i ts         -> Constr ann i (mkLamApp <$> ts)
  387           Case ann scrut branches -> Case ann (mkLamApp scrut) (mkLamApp <$> branches)
  388       | otherwise = t
  389       where
  390         currPath = fst (termAnn t)
  391 
  392 -- | Generate a fresh variable for the common subexpression.
  393 mkCseCandidate ::
  394   forall uni fun ann m.
  395   (MonadQuote m) =>
  396   Term Name uni fun (Path, ann) ->
  397   m (CseCandidate uni fun ann)
  398 mkCseCandidate t = CseCandidate <$> freshName "cse" <*> pure (void t) <*> pure t
