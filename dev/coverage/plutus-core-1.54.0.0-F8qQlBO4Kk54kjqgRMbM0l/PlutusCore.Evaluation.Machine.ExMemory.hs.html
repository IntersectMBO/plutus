   25 {- Note [Memory Usage for Plutus]
   26 
   27 The base unit is 'ExMemory', which corresponds to machine words. For primitives,
   28 we use static values for the size, see the corresponding instances. For
   29 composite data types, the Generic instance is used, + 1 for the constructor tag.
   30 For ADTs, the currently selected branch is counted, not the maximum value.
   31 Memory usage of the annotation is not counted, because this should be
   32 abstractly specifiable. It's an implementation detail.
   33 
   34 -}

   36 {- Note [Integer types for costing]
   37 We care about the speed of our integer operations for costing, this has a significant effect on speed.
   38 But we also need to care about overflow: the cost counters overflowing is a potential attack!
   39 
   40 We have a few choices here for what to do with an overflow:
   41 - Don't (this is what 'Integer' does, it's unbounded)
   42 - Wrap (this is what 'Int'/'Int64' and friends do)
   43 - Throw an overflow error (this is what 'Data.SafeInt' does)
   44 - Saturate (i.e. return max/min bound, this is what 'Data.SatInt does)
   45 
   46 In our case
   47 - Not overflowing would be nice, but 'Integer' is significantly slower than the other types.
   48 - Wrapping is quite dangerous, as it could lead to us getting attacked by someone wrapping
   49 their cost around to something that looks below the budget.
   50 - Throwing would be okay, but we'd have to worry about exception catching.
   51 - Saturating is actually quite nice: we care about whether `a op b < budget`. So long as `budget < maxBound`,
   52   then `a op b < budget` will have the same truth value *regardless* of whether the operation overflows and saturates,
   53   since saturating implies `a op b >= maxBound > budget`. Plus, it means we don't need to deal with
   54   exceptions.
   55 
   56 So we use 'Data.SatInt', a variant of 'Data.SafeInt' that does saturating arithmetic.
   57 
   58 'SatInt' is quite fast, but not quite as fast as using 'Int64' directly (I don't know
   59 why that would be, apart from maybe just the overflow checks), but the wrapping behaviour
   60 of 'Int64' is unacceptable.
   61 
   62 One other wrinkle is that 'SatInt' is backed by an 'Int' (i.e. a machine integer
   63 with platform-dependent size), rather than an 'Int64' since the primops that we
   64 need are only available for 'Int' until GHC 9.2 or so. So on 32bit platforms, we
   65 would have much less headroom.
   66 
   67 However, we don't build on 32bit platforms anyway, so we can ignore that.
   68 -}

   70 -- See Note [Integer types for costing]
   71 -- See also Note [Budgeting units] in ExBudget.hs
   72 type CostingInteger = SatInt

   74 -- | Counts size in machine words.
   75 newtype ExMemory = ExMemory CostingInteger
   76   deriving stock (Eq, Ord, Show, Generic, Lift)
   77   deriving newtype (Num, NFData, Read, Bounded)
   78   deriving (FromJSON, ToJSON) via CostingInteger
   79   deriving Serialise via CostingInteger
   80   deriving anyclass NoThunks
   81 instance Pretty ExMemory where
   82     pretty (ExMemory i) = pretty (unSatInt i)
   83 instance PrettyBy config ExMemory where
   84     prettyBy _ m = pretty m

   86 {- Note [Manual Semigroup and Monoid instances for Sum monoids]
   87 We don't do
   88 
   89     deriving via Sum A Semigroup A
   90     deriving via Sum A Monoid A
   91 
   92 because the 'Semigroup' and 'Monoid' instances for 'Sum' don't have @INLINE@ pragmas resulting in
   93 '<>' and 'mempty' appearing in the generated Core.
   94 
   95 So instead we implement @Semigroup A@ and @Monoid A@ instances manually for a 'Sum'-like @A@.
   96 -}
   97 
   98 -- See Note [Manual Semigroup and Monoid instances for Sum monoids].
   99 instance Semigroup ExMemory where
  100     (<>) = (+)
  101     {-# INLINE (<>) #-}
  102 
  103     stimes n mem = fromIntegral n * mem
  104     {-# INLINE stimes #-}
  105 
  106 -- See Note [Manual Semigroup and Monoid instances for Sum monoids].
  107 instance Monoid ExMemory where
  108     mempty = ExMemory 0
  109     {-# INLINE mempty #-}

  111 -- | Counts CPU units in picoseconds: maximum value for SatInt is 2^63 ps, or

  115   deriving newtype (Num, NFData, Read, Bounded)
